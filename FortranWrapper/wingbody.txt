      DESCRIPTION OF INPUT DATA FILE

The program was designed to be self contained, with the geometry
generator integral with the aerodynamic module.
The goal is to define the geometry with a small number of parameters
and compute the intermediate grid points internally.
The definition of common shapes for both airfoil sections and bodies
should allow many typical configurations from wind tunnel tests
and preliminary design studies to be defined quickly and accurately.

The configuration consists of one fuselage (optional) and a number
of wing segments that make up a reasonably complex wing and tail.
Each wing segment is trapezoidal, with root and tip parallel to
the x-axis.
The fuselage may be defined with a table of radius and camber at
each fuselage station, or as a nose-cylinder-afterbody of a small
set of pre-defined aerodynamic shapes.

The configuration geometry and the flow properties are defined
with NAMELIST statements.
There are four namelists defined in the program.
Two define the geometry of the configuration and two define
flow properties.

The fuselage is defined by the BODY namelist.
There is at most one fuselage, so there can only be one BODY input, and
it must come first.
The fuselage has circular cross-section at each point, but may
be cambered.

The BODY namelist is followed by one or more WING namelists.
Each WING record defines a trapezoidal wing region.
Many WING inputs may be used to build up a complex wing planform.
Tails and canards may be defined by WING records.

The flow properties are defined on the AERO namelist card.
The program allows one to compute the camber and twist required
to support a given lift distribution or the inverse problem
of computing the lift distribution on a given camber shape.
The shape of the wing of least drag for a given lift
coefficient may be computed, although the drag calculation
at subsonic speeds is not accurate enough to permit this
result to be of any great usefulness.

In addition to the namelist data records, any number of additional comment
records may be included to help keep track of the purpose of the data and
perhaps the identity of the creator.
Each record is read into the title buffer and therefore the last
non-namelist record is used as the title field on the output dataset.
It is possible to insert comments and new title records between the
namelists. The coding technique is quite simple. Each input record is
read into a temporary buffer and examined. If it begins with one of the
namelist tokens &BODY, &WING, &AERO, or &INCR, then the input file is
backspaced and control is passed to the appropriate input statement.
If not, then the record is copied to the title variable.


The next section describes each data item in the namelists
and gives the default value, if any.

The BODY namelist describes the body definition and panelling. Columns
2 through 6 contain the characters &BODY followed by one or more blanks.
The remainder of the input record contains data to be assigned to each
data item to be input. For example , LNOSE=16.4,.
Many of the data items are REAL, but if the input is a round number,
the decimal is optional.
If more data is to be input than will fit on one line, additional
lines may be used; no continuation mark is required.
The end of the data is signified by the end of namelist marker.
Most f77 systems use & or &END to indicate the end of a namelist,
but the correct way to do this in standard Fortran is to use a
slash symbol (/) as the terminator.

NAMELIST /BODY/
    BCODE  = body definition code. This determines the formula used to
             compute the shape of the nose and tail regions of the fuselage.
             =0  parabolic
             =1  Sears-Haack (l-V)
             =2  von Karman ogive
             =3  ellipsoidal
             =4  conical
             = -1 to allow user to input tables of body shape
                       (default value =0)
  The following quantities are only used if BCODE >=0
    LNOSE  = length of the nose region (default=0)
    LBODY  = overall length of the body (default=1)
    LTAIL  = length of the tail or afterbody region (default=0)
    RNOSE  = radius at the beginning of the nose. This is usually 0,
                but not always (F86 or Mig15) (default=0)
    RADIUS = radius of the cylindrical part of the body (default=1)
    RBASE  = radius at the end of the tail (default=0)
    ZNOSE  = z-coordinate (above body axis) of the nose  (default=0)
    ZBASE  = z-coordinate (above body axis) of the nose  (default=0)
    XNOSE  = x-coordinate of the nose (default=0)

  The following quantities are only used if BCODE < 0
    NTAB   = the number of entries in the following tables (max=51)
    X      = an array of x-coordinates of the body definition table
    R      = an array of corresponding radii
    Z      = an array of corresponding z-coor of the centers
                     (defaults to all zeroes)
    RPRIME = an optional array of dr/dx at each x.  If not input,
             this array will be computed by differencing. If you can
             compute the values, the result will be more accurate than
             relying on differences.

  The following quantities are used for all values of BCODE
    BPCODE = body panelling code  (default=0)
           = 0 to make panel edges that divide the region from XSTART
             to the end of the body into NROWS equal intervals
           = 1 to use the values in array BPFS
    BPFS   = array (length=NROWS+1) of body panelling fuselage stations
             that define the x-coordinates of the body panels if BPCODE=1
    CBAR   = reference length for pitching moment
    MACH   = Mach number
    NB     = number of strips used to panel a quadrant of the body
    NXBODY = number of fuselage definition points to define
             the line singularities.
    NROWS  = number of rows of panels used to panel body
    OC     = output control. Prints the given data plus that for
             all lesser values of OC
             =0 Coeff. of lift,drag,moment on wing,body, & wing-body
             =1 Pressure, Mach #, camber on each wing panel
                Pressures along meridians of body
             =2 Physical dimensions of body, area, volume
                Wing geometry, area, volume. Root,tip,span divisions
                Panel corner points, centroid, area, dihedral
             =3 Velocities induced by line singularities
             =4 Breakdown of velocities induced by each type of
                  line or surface element
             =5 Complete aerodynamic matrices
             =6 Reduced (transformed) matrices.

    OPT    = 0 to NOT compute the optimization matrix
    PER    = chordwise location
    RNL    = unit Reynolds number used in skin friction
             calculations; set to zero to suppress this calc.
    REFMOM = moment reference point for pitching moment calc.
    SREF   = reference area for coefficients.
    XSTART = starting value for panelling the body


  The variables NROWS and NB control the number of panels on the body.
  There will be NROWS panels dividing the region from x=XSTART to
  x=XNOSE+LBODY and NB panels dividing each quadrant (0 to 90 and
  90 to 180). Hence, there will be 2*NB*NROWS panels on the fuselage.
  The maximum number of panels on the body is 100.

  To repeat, since this is a FAQ (frequently asked question),
  if BCODE >=0, then you do not need to input anything for
  NTAB,X,R,RPRIME and Z. If you do, they will be ignored.
  Similarly, if BCODE < 0, you do not need to input anything
  for LB,LN,LT,RNOSE,RADIUS,RBASE,ZNOSE,ZTAIL, or XNOSE.
  If you do, they will be ignored.


NAMELIST/WING/
    ROOTLE = x-coordinate of the root leading edge
    ROOTTE = x-coordinate of the root trailing edge
    ROOTY  = y-coordinate of the root
    ROOTZ  = z-coordinate of the root
    TIPLE  = x-coordinate of the tip leading edge
    TIPTE  = x-coordinate of the tip trailing edge
    TIPY   = y-coordinate of the tip
    TIPZ   = z-coordinate of the tip
    F      array (length ROWS+1) of x-coordinates showing how the
              root chord is divided. (only when type=1 or 2)
    G      array (length ROWS+1) of x-coordinates showing how the
              tip chord is divided. (only when type=1 or 2)
    P      array (length COLS+1) of y-coordinates showing how the
              span is divided.  (only when type=1 or 3)
    SHEAR  array (length ROWS+1) of z-coordinates showing how the
              span is divided. (only when type=1 or 3)

    COLS                 = number of columns of panels in this region
    ROWS                 = number of rows of panels in this region
    TYPE   = code specifying whether root and tip and span are
                subdivided equally or according to user specification
           =1 user specifies all subdivisions; input MUST include
              F,G,P,SHEAR. Variables ROOTLE,ROOTTE,ROOTY,ROOTZ,
              TIPLE,TIPTE,TIPY,TIPZ are ignored.
           =2 user specifies root and tip subdivision and the span is
              evenly divided. Input must include F and G, as well as
              ROOTY,ROOTZ,TIPY, and TIPZ. Variables ROOTLE,ROOTTE,
              TIPLE,TIPTE,P and SHEAR are ignored.
           =3 user specifies span subdivision and the root and tip are
              evenly divided. Input must include P and SHEAR, as well as
              ROOTLE,ROOTTE,TIPLE, and TIPTE. Variables ROOTY,ROOTZ,
              TIPY,TIPZ,F and G will be ignored.
           =4 root,tip and span are evenly divided. Input must include
              ROOTLE,ROOTTE,ROOTY,ROOTZ,TIPLE,TIPTE,TIPY, & TIPZ.
              Variables F,G,P,SHEAR are ignored.
    OC     = output control (see /BODY/ for details)

    MACH   = free stream Mach number
    SREF   = wing reference area
    REFMOM = moment reference point
    CBAR   = reference chord length
    OPT    = 0 to NOT compute the optimization matrix
    SECT   = code defining the shape of the airfoil section
            = 0 parabolic arc
            = 1 double wedge
            = 2 30-70 hex
            = 3 wedge (blunt base)
            =-1 NACA 000x
            =-2 NACA 6400x
            =-3 NACA 6500x
            =-4 RAE 101
    TCROOT  = thickness/chord at the root of this wing region
    TCTIP   = thickness/chord at the tip of this wing region
    RNL     = Reynolds number per foot of the flow (0 for no friction drag)
    PER     = relative location of the control points  as fraction of
                chord thru centroid of panel. (Usual rule is to
                use 0.95 for supersonic cases and 0.85 for subsonic)
                (default = 0.95)

NAMELIST/AERO/
    CASE   defines the problem for which a solution is sought
            =1  compute CAMBER for a given table of CP (delta-Cp)
            =2  compute CP (upper and lower) for a given CAMBER
            =3  compute CAMBER for CL=CLBAR and least drag
            (default=2)

    BODYCPEQ = the formula used to compute pressure on the
                  body from velocity components:
            =0 linear
            =1 slender body
            =2 2nd order
            =3 isentropic
            (default=1)

    WINGCPEQ = the formula used to compute pressure on the
                  wing from velocity components:
            =0 linear
            =1 slender body
            =2 2nd order
            =3 isentropic
            (default=0)

    BDYALP  = body angle of attack (degrees) when the
                    configuration is at zero angle of attack.
                    (default=0)
    CAMBER  = array (length=# of wing panels) of values of dz/dx
    CP      =
    CLBAR   = constrained value of wing lift coefficient (for CASE=3)

    SREF    = wing reference area (total of left & right)
    CBAR    = reference chord length
    REFMOM  = x-coordinate of the moment reference point
    OC      = output control (see /BODY/ for details)

NAMELIST/INCR/
    DELTA  = change in angle of attack applied to whole configuration
    OC     = output control (see /BODY/ for details)




          DESCRIPTION OF THE OUTPUT FILE wingbody.out

The file wingbody.out contains the output from the program.
The actual contents of this file are under the control of the
variable OC, which controls which quantities are printed.
The default value of OC=1 prints the pressures, Mach number,
and camber on each wing panel and the pressures along meridian
lines on the body, as well as the integrated forces and moments
on the components and complete configuration.

The panels are numbered from leading edge to trailing edge and
then from root to tip. Setting OC to 2 will give a complete
description of the panel geometry.
